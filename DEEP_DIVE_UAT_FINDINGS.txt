Issue #3: LOADING STATE UX — ACCEPTABLE WITH CAVEAT
────────────────────────────────────────────────────────────────────────────────
Severity: MEDIUM (User Experience)
Component: fetchDeepDiveData loading indicator
Viewport: All
Current Code: 5-second safety timeout, cancels spinner but not fetch

Problem:
  If Supabase is slow (>5s), spinner disappears and user sees blank panel.
  Fetch may still be ongoing. When data arrives (10s+), it appears without
  transition or explanation.

Expected:
  Clear feedback about load state: either continues spinning, shows estimated
  time, or fetches are aborted with explanatory message.

Current:
  Spinner disappears; if data arrives later, displays seamlessly (actually
  UX is OK, just unclear what happened).

Acceptable Because:
  • Rare edge case (Supabase should be <1s typical)
  • When data arrives, it displays correctly
  • Better than infinite spinner
  • User can refresh or navigate away

Recommended Enhancement (Optional):
  • Show "Loading took longer than expected. Checking again..." message
  • OR increase timeout to 10 seconds
  • OR use AbortController to cancel fetch after timeout

Implementation Effort: 10 minutes

BLOCKING: No, acceptable as-is.
