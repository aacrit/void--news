What runs automatically — not your job

The ig-pipeline.yml workflow generates one draft a day, renders the slide images, writes the caption with Gemini flash-lite, and on Mon/Thu publishes whatever you have approved to Instagram, then cross-posts it to Bluesky. The token refreshes itself weekly and metrics collect daily. You never render, caption, or publish by hand. You decide what ships.

01

Accounts & Meta setup

one-time

Sign up with a personal phone number (Meta bans virtual numbers). If void.news is taken, fall back to voidnews or void_news — hyphens are blocked, so the double-dash wordmark is impossible.

Threads picks up the IG handle automatically. Reserve the matching handle on Bluesky and X even if unused — it blocks impersonation.

IG app → Settings → Account → Switch to Professional → Creator. In-app only; there is no API for this.

The "Professional dashboard" entry appears under your bio.
Same stories. Same order. For everyone.
6-axis bias on every article, not every outlet.
No signup. No tracking. Free forever.

Link: void-news.pages.dev. Photo: brand mark at 320×320, ink on ivory. The Graph API can't edit bio or photo, so this stays manual forever.

Meta requires a real-person admin via Facebook. Create a Page at facebook.com/pages/create ("void --news", News & Media), then IG → Settings → Linked Accounts → Facebook → that Page.

Both assets show under the same Page in Meta Business Suite.

At developers.facebook.com: a Business app, then Add Product → Instagram Graph API + Instagram Messaging. Note the App ID and App Secret.

In the Graph API Explorer, request: instagram_basic, instagram_content_publish, instagram_manage_comments, instagram_manage_messages, instagram_manage_insights, pages_show_list, pages_read_engagement. Exchange the short-lived token for a 60-day one — that is INSTAGRAM_GRAPH_TOKEN. The weekly cron rotates it from here on.

curl ".../me?access_token=$TOKEN" returns your IG user object.

Query /me/accounts for your Page id, then /{PAGE_ID}?fields=instagram_business_account. The numeric instagram_business_account.id is INSTAGRAM_USER_ID.

Bluesky → Settings → App Passwords → Add. Save it as BLUESKY_APP_PASSWORD; BLUESKY_HANDLE is the full handle. Never use your main password.

META_WEBHOOK_VERIFY_TOKEN (any 32+ char string) and IG_ADMIN_PASSWORD (basic-auth gate on /admin/ig). Generate with python -c "import secrets; print(secrets.token_urlsafe(32))".

02

Plant the secrets

one-time

At repo → Settings → Secrets → Actions:

INSTAGRAM_GRAPH_TOKEN   INSTAGRAM_USER_ID
META_APP_ID             META_APP_SECRET
BLUESKY_HANDLE          BLUESKY_APP_PASSWORD
IG_SECRET_WRITE_TOKEN   IG_ADMIN_USER
IG_ADMIN_PASSWORD

IG_SECRET_WRITE_TOKEN is a fine-grained PAT scoped to this repo with Secrets: Read & Write (the token-refresh cron writes the rotated token back). GEMINI_API_KEY and the Supabase secrets already exist.

All secrets appear in the Actions secrets list.

For the webhook function at /api/ig/webhook — CF dashboard → Pages → void-news → Settings → Environment variables (Production): META_APP_SECRET, META_WEBHOOK_VERIFY_TOKEN, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY. Redeploy after saving (CF won't pick up env changes otherwise).

cp .env.example .env and fill in IG_ADMIN_USER, IG_ADMIN_PASSWORD, SUPABASE_SERVICE_ROLE_KEY, NEXT_PUBLIC_SUPABASE_*. You run the admin page locally to approve posts.

npm run dev → localhost:3000/admin/ig accepts your basic-auth login.
03

Go live & pass review

one-time

Open the PR from claude/social-media-content-check-di1uzv, read the diff, merge. This deploys the pipeline, the admin page, and the /privacy + /press pages, and activates the daily/publish crons.

/privacy and /press load, and the webhook returns 403 to a bad verify token (proves it deployed).

Owned domains → add void-news.pages.dev → DNS TXT method → add the record at your DNS provider → Verify.

60–90s: show a draft in /admin/ig, its slides + caption, click Approve, run a dry-run publish, show the post on Instagram. Submit App Review for instagram_content_publish, instagram_manage_comments, instagram_manage_messages, instagram_manage_insights, each with the screencast, a one-paragraph use case, and the privacy URL. Approval takes 1–7 business days.

App → Webhooks → Instagram. Callback https://void-news.pages.dev/api/ig/webhook, paste your verify token, subscribe to comments, mentions, messages, message_reactions, media.

Each field shows "Subscribed", no red banners.
04

Launch day

one-time

Actions → IG Pipeline → Run workflow → mode seed-launch. It generates rows 1–10, renders every slide, and writes captions, then parks them in draft for you.

Ten rows tagged launch #1–#10 appear in /admin/ig.

In /admin/ig, scrub the slides and read each caption: no em dashes, no "notable / significant / interestingly", numbers do the work. Edit inline if needed, then save + approve. This editorial yes/no is the one gate that never automates.

Run mode publish to ship the approved opener (or wait for the next Mon/Thu cron). The same morning, post the "something I'm working on" version from your personal account to your network.

Post 1 is live on Instagram and mirrored to Bluesky.
05

Ongoing rhythm

recurring

The cron drops a fresh, captioned draft each morning. Open /admin/ig, give it a yes or no. Approved posts ship on the next Mon/Thu; nothing publishes without your approval.

/admin/ig/inbox collects the DMs, comments, mentions, and hashtag leads the pipeline flagged for a human. Auto-replies only cover trivial cases; anything with a question or a real person is yours to answer.

There is no LinkedIn automation. For a post worth long-form, copy the caption and post it manually in your own voice.

Weekly, skim the metrics the pipeline collected (reach, saves, follows). Monthly, consider one Reel — the only format the automation doesn't produce.
